### STEP 1：Perplexity APIの取得と初期設定

まずは、PerplexityのAPIキーを取得します。
これ、普通のPerplexity Pro（有料プラン）に入っていなくても、API用のクレジットを少しチャージするだけで誰でも使えます。しかも、1回のリクエストあたりの料金は数円程度なので、月額5万円の案件なら経費はほぼ誤差です。

1. **PerplexityのAPI設定画面にアクセスする**
   ブラウザで「Perplexity API Settings」（https://www.perplexity.ai/settings/api）にアクセスして、あなたのアカウントでログインしてください。

2. **クレジットカードを登録してチャージする**
   「Billing」タブをクリックして、クレジットカード情報を登録します。最初は最低金額の「$10（約1,500円）」をチャージしておけば十分です。これだけで何百記事も生成できます。

3. **APIキーを発行する**
   「API Keys」の項目にある「Generate API Key」というボタンをクリックします。
   すると、`pplx-xxxxxxxx...` のような長い文字列が表示されるので、これをコピーしてメモ帳などに貼り付けておいてください（一度画面を閉じると二度と表示されないので注意です）。

これでPerplexity側の準備は完了です。

### STEP 2：WordPressの「アプリケーションパスワード」を発行する

次に、外部ツール（Make）からあなたの（またはクライアントの）WordPressに記事を自動投稿できるようにするための設定をします。
セキュリティを担保しつつ連携するために、「アプリケーションパスワード」というものを発行します。

1. **WordPressの管理画面にログインする**
2. 左メニューの「ユーザー」＞「プロフィール」をクリックします。
3. 画面を一番下までスクロールすると、「アプリケーションパスワード」という項目があります。
4. 「新しいアプリケーションパスワード名」の入力欄に、分かりやすく「`Make-AutoPost`」などと入力します。
5. 「新しいアプリケーションパスワードを追加」ボタンをクリックします。
6. 画面に `xxxx xxxx xxxx xxxx xxxx xxxx` のような、4文字ごとにスペースで区切られたパスワードが表示されるので、これも必ずコピーしてメモ帳に控えてください。

※注意：WordPressのプラグインで「セキュリティ系」のものを入れていると、外部からのAPI接続がブロックされることがあります。もし後述の連携でエラーが出る場合は、一時的にセキュリティプラグイン（SiteGuardなど）の設定を確認してみてください。

### STEP 3：Make（旧Integromat）で自動化シナリオを構築する

ここが一番の肝になります。
自動化ツール「Make」を使って、PerplexityとWordPressを繋ぎます。
Makeは無料プランの範囲内で十分に動かせます。

1. **Makeに会員登録・ログインする**
   Make（https://www.make.com/）にアクセスして、アカウントを作成してください。

2. **新しいシナリオを作成する**
   ログイン後、右上の「Create a new scenario」をクリックします。

3. **トリガー（起動タイミング）を設定する**
   画面中央の大きな「＋」ボタンをクリックし、検索窓に「**Tools**」と入力して、Toolsモジュールを選択します。
   その中から「**Basic trigger**」を選択します。
   ※毎日自動で実行させたいので、モジュールの左下にある「時計マーク（Schedule setting）」をクリックし、以下のように設定します。
   - Run scenario: `Every day`
   - Time: `08:00`（朝の8時に自動実行されるようにします）

4. **Perplexity APIを呼び出すHTTPモジュールを追加する**
   Toolsモジュールの右側にある「Add another module」をクリックし、検索窓に「**HTTP**」と入力します。
   緑色の「HTTP」アイコンを選択し、アクションの中から「**Make a request**」を選びます。

   ここが一番間違えやすいので、以下の設定値を一言一句間違えずに正確に入力してください。

   - **URL**: `https://api.perplexity.ai/chat/completions`
   - **Method**: `POST`
   - **Headers**:
     2つのヘッダーを追加します。「Add item」を2回クリックして、以下のように入力してください。
     - 1つ目:
       - Name: `Authorization`
       - Value: `Bearer [STEP1で取得したPerplexityのAPIキー]` （※Bearerとキーの間には半角スペースを1つ入れてください）
     - 2つ目:
       - Name: `Content-Type`
       - Value: `application/json`
   - **Body type**: `Raw`
   - **Content type**: `JSON (application/json)`
   - **Request content**:
     以下のコードをそのままコピーして、貼り付けてください。

```json
{
  "model": "sonar",
  "messages": [
    {
      "role": "system",
      "content": "あなたはプロのIT・テック業界のニュースライターです。指定されたテーマに関する最新の重要ニュースをネット上から検索し、読者が興味を持つような1つのまとめ記事を作成してください。出力は必ず以下のルールを厳守してください。\n1. 記事タイトルは、読者の目を引くキャッチーなものにすること。\n2. 本文はHTML形式（h2, h3, pタグを使用）で、論理的かつ分かりやすく書くこと。\n3. 必ず記事の最後に「参考ソース」として、検索したニュースの実際のURLをリンク形式（<a href=\"URL\" target=\"_blank\">サイト名</a>）で3つ以上記載すること。\n4. ハルシネーション（嘘の情報）は一切排除し、事実に基づいた内容にすること。"
    },
    {
      "role": "user",
      "content": "「生成AIのビジネス活用における最新トレンドと企業の導入事例」について、過去24時間以内の最新ニュースを検索し、要約記事を作成してください。出力は、JSON形式ではなく、記事のタイトルを1行目に書き、2行目以降にHTMLの本文を続けてください。タイトルと本文の区切りには「---SPLIT---」を入れてください。"
    }
  ],
  "temperature": 0.2
}
```

※「model」には、Perplexityの最新検索モデルである `sonar` を指定しています。これで常に最新のWeb検索結果を反映してくれます。
※「user」の内容（上記の「生成AIのビジネス活用〜」の部分）は、クライアントの業界に合わせて自由に変えてみてください。不動産業界なら「不動産テックや住宅業界の最新動向」、医療業界なら「医療DXやヘルスケアITの最新ニュース」といった感じです。

5. **WordPressモジュールを追加する**
   HTTPモジュールの右側に「Add another module」をクリックし、検索窓に「**WordPress**」と入力します。
   「**Create a Post**」を選択します。

   ここで、WordPressとの接続設定（Connection）を行います。
   - **Connection**: 「Add」をクリックします。
   - **Credentials**: `Basic Auth`（またはWordPress Username and Password）を選択します。
   - **WordPress URL**: クライアントのWordPressのトップURL（例: `https://example.com` ※最後のスラッシュは不要です）を入力します。
   - **Username**: WordPressのログインユーザー名を入力します。
   - **Password**: STEP2で発行した「アプリケーションパスワード（スペース含む24文字）」を入力します。

   接続が成功すると、WordPressの投稿設定画面が表示されます。

   ここで、Perplexityから返ってきたテキストを「タイトル」と「本文」に切り分ける必要があります。
   Makeの「Text Parser」モジュールを間に挟むと綺麗に分けられますが、一番簡単な方法として、今回はPerplexityの出力をそのまま本文（Content）にマッピングし、タイトルは固定にするか、手動で後から調整する形にするか、あるいは以下の設定を「Content」に入力してください。

   HTTPモジュールから返ってくるデータは、以下の変数で取得できます。
   `data.choices[1].message.content`

   これを、WordPressモジュールの「Content」の部分にドラッグ＆ドロップ（またはクリックして選択）してマッピングします。
   - **Title**: 「最新の業界特化ニュースまとめ（自動生成）」などとするか、Perplexityの出力から抽出します。
   - **Status**: ぶっちゃけ、最初は「**Draft（下書き）**」にしておくのが一番安全です。クライアントが最終確認して、ボタン一つで「公開」にできるようにしておけば、ハルシネーションによる事故を100%防げます。これだけでクライアントの安心感がまじで違います。

6. **テスト実行をする**
   Makeの画面左下にある「Run once」というボタンをクリックしてみてください。
   ぐるぐると処理が走り、すべてのモジュールに緑色のチェックマーク（✔）がつけば成功です。
   実際にWordPressの「投稿一覧」を開いて、下書きに綺麗なニュース記事が保存されているか確認してみてください。
