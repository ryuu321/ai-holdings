# note記事生成スキル｜チビの哲学

## 記事生成の前に確認する3つ
1. **この記事で読者の何が変わるか**（曖昧ならテーマを絞り直す）
2. **チビの哲学にしか書けない実体験があるか**（なければボツ）
3. **無料部分の「引き」が1行で言えるか**（言えないなら設計が甘い）

## インプット質問票

### Q1. テーマと核心メッセージ（1文で）
このテーマで一番言いたいことを一文で。

```
テーマ：
核心メッセージ：
```

### Q2. 自分の実体験（数字・場面・セリフあり）
```
エピソード：
```

### Q3. 読者のあるある場面（3つ）

```

③：
```

```

```

## 無料部分（300字）の設計【最重要】

有料転換はここで決まる。3つの型から選ぶ。

### 型A：あるある共感型（最も汎用）
```
[場面描写1行] + [場面描写1行] + [場面描写1行]
（読者が「俺のことだ」と思う3連打）

「〇〇だから」って思ってませんか。

違うんです。[核心メッセージを一言で]。

[自分の実体験を1行]

続きで話します。
```

### 型B：問い投げかけ型
```
[問いかけ1行]

ほとんどの人は[誤解されてること]だと思ってる。

でも実際は違う。[核心メッセージ]。

僕が[実体験]で気づいたことを話します。
```

### 型C：断言型（インパクト重視）
```
[断言する一文。短く。]

[その理由を1行]

[あるある場面を1行]

[自分の立場を1行：身長155cmでも〜]

続きに全部書きました。
```

**引きのNGワード**：「〜してみてください」「ぜひ〜」「参考になれば」

## 有料部分の構成（2,700字）

### ブロック①：問題の深掘り（800字）
**目的**: 読者に「自分のことが言語化された」と感じさせる

```
[あるある場面を3〜4パターン、200字ずつ細かく描写]
（場面・感情・行動の順で書く）

「これ、全部[根本原因]からきてるんですよ。」で締める

[心理的な根拠を1〜2段落で補強]
（人間の本質として普遍化する）
```

感情の流れ：「共感」→「言語化された驚き」→「なぜ？という疑問」

### ブロック②：解決策（800字）
**目的**: 「難しくない、できる」と思わせる

```
「難しく考えなくていいんです。」で入る

[解決策①を具体的に。キーワードは**太字**]
（抽象論→具体例→もう一度抽象論の順）

[解決策②]

[解決策③]

「これが[テーマの本質]の正体です。」で締める
```

感情の流れ：「疑問」→「なるほど」→「自分にもできそう」

### ブロック③：実践・対比（500字）
**目的**: 頭の中の変化を「見える化」する

```
「じゃあ実践の話をします。」で入る

コードブロックで対比を見せる：
---
依存している状態の頭の中
→ [具体的な思考パターン]
→ [具体的な思考パターン]

解決後の頭の中
→ [具体的な思考パターン]
→ [具体的な思考パターン]
---

[1〜2行のコメント]
```

感情の流れ：「わかる」→「自分の頭の中と一致」→「変わりたい」

### 締め（600字）
**目的**: 記事全体を一つのメッセージに凝縮する

```
「最後に、一番伝えたいことを話します。」で入る

[Q5の核心メッセージを展開。太字で一文]

[比較をやめることの重要性、または等身大でいることへの言及]

[自分の実体験で締める（身長・実体験など）。読者への励ましではなく、事実として淡々と]

「次号に続く。」
```

感情の流れ：「ハッとする」→「腑に落ちる」→「また読みたい」

## 文体チェックリスト（生成後に確認）

- [ ] 語尾が「〜んですよね」「〜んです」「〜ですよ」になっているか
- [ ] 一文が40字以内か（超えていたら分割）
- [ ] 重要な一文が**太字**になっているか
- [ ] 「〜しましょう」「〜してください」が使われていないか
- [ ] 無料部分が300字以内か
- [ ] 各ブロックに実体験か具体的場面があるか
- [ ] 「次号に続く。」で終わっているか

## タイトルの型
```
「[核心メッセージ]。── [サブタイトル]」
```
- 例：「自信がないから、依存する。── 自己肯定感を上げるって、そういうことじゃないんですよ。」
- 例：「嫌われたくないから、好かれない。── ニブイチの精神の話をします。」
- 例：「比較が、自信を殺す。── 等身大でいることの、本当の意味。」
