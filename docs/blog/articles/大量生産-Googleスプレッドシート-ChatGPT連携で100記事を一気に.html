ぶっちゃけ、最初は「3,000円のドブ捨て」から始まりました

まず、これを語る前に僕の恥ずかしい失敗談を聞いてください。 先月の15日、水曜日の夜のことなんですけど。「スプレッドシートと連携すれば爆速じゃん！」って気づいて、浮かれて見よう見まねで設定してみたんですよね。

その時、調子に乗って200記事分くらいのキーワードをスプレッドシートに並べて、一気に実行ボタンを押したんです。でも、プロンプトの書き方が悪かったのか、APIの設定ミスなのか、返ってきたのは「Error」の文字と、なぜかアラビア語みたいな謎の文字列だけ。

しかも、OpenAIの管理画面を見たら、その数分だけでAPI使用料が3,000円くらい飛んでたんですよ。まじで泣きそうになりました。筋トレ1時間やって忘れたようとしたけど、無理でしたね。

でも、そこから「何がダメだったのか」を徹底的に調べたんです。 原因は、スプレッドシートから送る際の「トークン制限」と、一気にリクエストを送りすぎてAPI側でエラーを吐かれていたことでした。

ここからは、そんな僕の失敗を皆さんが踏まないように、最短ルートで「100記事一括生成」を成功させるためのワークフローを共有しますね。正直、これを知ってるだけで副業の作業時間は10分の1になります。

ステップ1：APIキーの取得と「絶対にやってはいけない設定」

まずはOpenAIの公式サイトからAPIキーを取得するんですけど、ここで初心者がやりがちなミスが「Usage Limit（利用制限）」を設定していないことなんですよね。

僕が3,000円を無駄にしたのは、このリミットを設定してなかったからです。 「Settings」の「Billing」から、月間の使用上限を5ドルとか10ドルに設定しておきましょう。これ、マジで大事です。保険みたいなものですね。

あと、APIキーは絶対に人に教えちゃダメですよ。スプレッドシートに直接書き込む方法もあるんですけど、僕はGoogle Apps Script（GAS）の中に隠すようにしています。

ステップ2：スプレッドシートを「自動執筆マシン」に変えるGASコード

ここが一番の肝なんですけど、ブラウザ版のChatGPTみたいに人間がポチポチ打つんじゃなくて、スプレッドシートに「GPT関数」を作っちゃいます。

「拡張機能」→「Apps Script」を開いて、コードを貼り付ける作業があるんですけど、最初はこれが全然うまくいかなくて。 ネットに落ちてる古いコードを使ったら、今のGPT-4o（オムニ）に対応してなくてエラー吐きまくったんですよね。

そこで、僕が改良に改良を重ねた、安定して動くコードの雛形を貼っておきます。

```javascript function GPT_ASK(prompt) { const apiKey = "ここにあなたのAPIキー"; const apiUrl = "https://api.openai.com/v1/chat/completions"; const payload = { model: "gpt-4o", messages: [ {role: "system", content: "あなたはプロのライターです。読者の心に刺さる文章を書いてください。"}, {role: "user", content: prompt} ], temperature: 0.7 }; const options = { method: "post", contentType: "application/json", headers: { Authorization: "Bearer " + apiKey }, payload: JSON.stringify(payload), muteHttpExceptions: true }; const response = UrlFetchApp.fetch(apiUrl, options); const json = JSON.parse(response.getContentText()); if (json.choices && json.choices.length > 0) { return json.choices[0].message.content; } else { return "エラーが発生しました: " + response.getContentText(); } } ```

これを保存すると、スプレッドシートのセルに `=GPT_ASK("ブログの導入文を書いて")` って打つだけで、回答が返ってくるようになります。これ、初めて動いた時は感動して、夜中に一人で「よっしゃ！」って叫びました（笑）。

ステップ3：100記事を一気に生成する「プロンプトの組み立て方」

さて、ここからが本題です。 単にキーワードを投げるだけだと、AIっぽい、どこかで見たような薄い記事しか出てこないんですよね。

僕はよくA列に「キーワード」、B列に「ターゲット」、C列に「構成案」を入れて、D列で一気に記事を書かせるようにしています。

例えば、こんな感じです： - A2：筋トレ　初心者　サプリ - B2：20代のガリガリな男性 - C2：1.なぜサプリが必要か 2.おすすめ3選 3.飲むタイミング

そしてD2のセルには、こんな感じの数式を入れます。 `=GPT_ASK(A2 & "というキーワードで、" & B2 & "に向けて、" & C2 & "の構成でブログ記事を3000文字程度で書いてください。口調は親しみやすく。")`

これを100行分下にドラッグするだけ。 これで、放置している間に100記事が勝手に生成されていきます。

先週の火曜日、ジムでベンチプレスを上げている間に、スマホでスプレッドシートを確認したら、100記事が全部完成してた時は「これぞ不労所得への第一歩だな」って思いました。

ステップ4：大量生産で陥る「地獄のエラー」を回避する方法

ただし、一気に100行ドラッグすると、高確率でエラーが出ます。 理由は「APIのレートリミット（短時間でのリクエスト制限）」です。

これを回避するために、僕は「10行ずつやる」か、GASのコードに `Utilities.sleep(1000)` を入れて、1秒ずつ実行を遅らせるようにしています。 正直に言うと、最初は焦って一気にやろうとして、全部のセルがエラーで真っ赤になったことがあります。あの絶望感はすごいので、皆さんはゆっくりやってくださいね。

あと、生成された記事は必ず「自分の目」でチェックするのが鉄則です。 たまにAIが嘘をついたり、日本語が変だったりすることもありますから。僕は生成された100記事をざっと読んで、変なところだけリライトするスタイルでやってます。これだけで、ゼロから書くより10倍速いです。
